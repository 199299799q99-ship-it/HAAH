AI 在执行任何指令时，必须严格遵守以下“读取-执行-测试-同步”的四步循环：
阶段一：前置准备 (Pre-task Requirement)
强制读取： 在开始每一项新任务（包括代码修改、新功能开发、Bug 修复）之前，必须首先完整读取项目根目录下的 README.md 文件。
深度对齐： 必须确保对当前的“代码结构”、“架构设计”和“技术栈”有深入理解，严禁在未理解全局结构的情况下直接编写代码。
阶段二：功能开发 (Development Phase)
规范执行： 按照 README 中定义的架构和技术栈编写代码。
结构一致性： 新增文件或模块必须符合 README 中定义的项目目录结构。
阶段三：功能测试 (Testing Phase)
规范执行： 使用chrome mcp  对开发的功能进行测试，确保符合预期。
阶段四：同步更新 (Post-task Requirement)
文档同步： 每当完成一项功能开发或对项目结构做出实质性调整后，必须立即同步更新 README.md 文件。
完整性检查： 更新后的文档必须反映最新的代码现状，确保文档与代码库高度一致。
